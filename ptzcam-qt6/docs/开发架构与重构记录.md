## 一、ControlCard 公共组件

### 1.1 解决的问题

四个视频 Tab（USB/RTSP/NDI/ONVIF）的 `_create_control_card()` 方法存在大量重复代码：
- QFrame 创建 + 样式设置
- QVBoxLayout 布局 + margins + spacing
- QLabel 右对齐 + fixedWidth
- QComboBox 统一宽度 + maxVisibleItems
- 最后 `setFixedHeight(120)`

每个 Tab 的 control_card 约 **90 行**，四个合计 **~370 行**，重构后约 **100 行**。

### 1.2 设计

```python
card = ControlCard()                    # 统一框架 + 120px高度
row = card.add_row()                     # 新行
row.addWidget(ControlCard.make_label("设备:"))  # 标准标签
self.combo = ControlCard.make_combo(220)        # 标准下拉
ControlCard.add_stretch(row)                    # 弹性空间
```

### 1.3 文件位置

`app/widgets/control_card.py` — `ControlCard` 类。

## 二、串口异步连接模式

### 2.1 背景

某些蓝牙虚拟串口（COM3/COM4）调用 `pyserial.Serial()` 会无限阻塞，导致 Qt 界面完全冻结。传统的 `thread.join()` 也会阻塞主线程。

### 2.2 解决方案

```python
# 1. 后台线程打开串口
def _do_connect():
    result[0] = controller.connect_serial(...)

thread = threading.Thread(target=_do_connect, daemon=True)
thread.start()

# 2. QTimer 轮询线程完成状态（不阻塞事件循环）
def _poll():
    if done[0]:
        _on_serial_done()
        return
    QTimer.singleShot(100, _poll)

QTimer.singleShot(100, _poll)

# 3. 独立的 5 秒超时定时器（存为实例变量防止 GC）
self._serial_timeout_timer = QTimer()
self._serial_timeout_timer.setSingleShot(True)
self._serial_timeout_timer.timeout.connect(_on_serial_timeout)
self._serial_timeout_timer.start(5000)
```

### 2.3 注意事项

- **QTimer 必须存为实例变量**（如 `self._serial_timeout_timer`），否则局部变量会被 Python GC 回收导致超时不触发
- `done[0]` 标记用于防止线程完成和超时同时触发 UI 更新
- 按钮状态通过 `_serial_pending` 标志防止重复点击

## 三、PyInstaller 编译注意事项

### 3.1 必须清缓存

每次编译前必须删除：
```
build/  dist/  __pycache__/  *.spec
```

否则旧代码会残留导致功能不生效。

### 3.2 Assets 路径

- 开发版：相对路径 `assets/xxx.svg`
- 编译版：`sys._MEIPASS/assets/xxx.svg`
- 样式表中引用 SVG 必须用**运行时解析的绝对路径**
- 在 Python 代码中动态构造路径传给 stylesheet

### 3.3 WSDL 文件

ONVIF 的 WSDL 文件在 `site-packages/wsdl/`，编译时需：
```
--add-data "path/to/wsdl;wsdl"
```

运行时通过 `sys._MEIPASS/wsdl/` 访问。

## 四、函数拆分原则

### 4.1 单函数最大长度

建议不超过 **50 行**。超过说明职责可能过多。

### 4.2 theme.py 拆分案例

`get_global_stylesheet()` 从 137 行的单一大函数拆分为 6 个职责清晰的子函数：

| 函数 | 职责 |
|------|------|
| `_bg_section()` | 背景图片/颜色 |
| `_base_widget_section()` | 基础控件样式 |
| `_combo_box_section()` | 下拉框+滚动条 |
| `_status_bar_section()` | 状态栏+列表 |
| `_tab_section()` | 标签页半透明样式 |
| `_size_grip_section()` | 右下角缩放柄 |

### 4.3 onvif_device.py 拆分案例

`connect()` 从 152 行拆分为：
- `_build_credentials_list()` — 构建凭据列表
- `_try_credential_pair()` — 尝试一组凭据
- `_read_device_info()` — 读取设备信息
- `_check_media_service()` — 检查媒体服务
- `_fetch_rtsp_url()` — 获取 RTSP URL

### 4.4 rtsp_capture.py 拆分案例

`run()` 从 111 行拆分为：
- `_setup_transport_env()` — 设置传输环境变量
- `_on_connect_failed()` — 连接失败处理
- `_read_frame_loop()` — 帧读取主循环
- `_on_stream_error()` — 流异常处理
